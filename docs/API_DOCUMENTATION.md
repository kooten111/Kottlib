# YACReader Library Server API Documentation

This document details the API endpoints for the YACReader Library Server, covering both v1 and v2 versions.

## Overview

The server uses a custom HTTP server implementation. Requests are routed based on regex patterns defined in `requestmapper.cpp`.
- **v1 API**: Primarily uses custom text-based formats (key-value pairs or delimited strings) and HTML for the web UI.
- **v2 API**: Introduces JSON for most metadata endpoints, but retains some custom text formats for specific operations (like comic opening).

### Endpoints

| Method | Path | Controller | Description |
| :--- | :--- | :--- | :--- |
| GET | `/` | `LibrariesController` | List available libraries (HTML) |
| GET | `/library/{id}/folder/{folderId}` | `FolderController` | List content of a folder (HTML) |
| GET | `/library/{id}/folder/{folderId}/info` | `FolderInfoController` | Get folder information (Custom Text) |
| GET | `/library/{id}/comic/{comicId}` | `ComicDownloadInfoController` | Get comic download info (Custom Text) |
| GET | `/library/{id}/comic/{comicId}/info` | `ComicController` | Get full comic info (Custom Text) |

| POST | `/sync` | `SyncController` | Sync reading progress |

### Data Formats (v1)

#### Folder Info (`/library/{id}/folder/{folderId}/info`)
Returns a list of comics and subfolders in a custom delimited format.
**Content-Type**: `text/plain; charset=utf-8`

**Format**:
```text
/library/{libId}/comic/{comicId}:{fileName}:{fileSize}
...
```
*Note: Subfolders are recursively serviced, but the output format for folders is not explicitly distinguished in the controller code examined, suggesting it might just list comics recursively or use a similar format.*

#### Comic Download Info (`/library/{id}/comic/{comicId}`)
**Content-Type**: `text/plain; charset=utf-8`

**Format**:
```text
fileName:{fileName}
fileSize:{fileSize}
```

#### Comic Info / Remote (`/library/{id}/comic/{comicId}/info` or `/remote`)
**Content-Type**: `text/plain; charset=utf-8`

**Format**:
```text
library:{libraryName}
libraryId:{libraryId}
previousComic:{prevId} (only if remote)
nextComic:{nextId} (only if remote)
{key}:{value}
...
```
The `{key}:{value}` pairs are generated by `ComicDB::toTXT()`.

### Data Formats (v2)

#### Libraries (`/v2/libraries`)
**Content-Type**: `application/json`

**Format**:
```json
[
  {
    "name": "Library Name",
    "id": 123,
    "uuid": "uuid-string"
  },
  ...
]
```

#### Folder Info (`/v2/library/{id}/folder/{folderId}/info`)
**Content-Type**: `text/plain; charset=utf-8`

**Format**:
```text
/v2/library/{libId}/comic/{comicId}:{fileName}:{fileSize}:{readStatus}:{hash}
...
```
*Note: `readStatus` is 1 if read, 0 otherwise.*

#### Comic Remote / Download (`/v2/library/{id}/comic/{comicId}/remote`)
**Content-Type**: `text/plain; charset=utf-8`

**Format**:
```text
library:{libraryName}
libraryId:{libraryId}
previousComic:{prevId} (only if remote)
previousComicHash:{prevHash} (only if remote)
nextComic:{nextId} (only if remote)
nextComicHash:{nextHash} (only if remote)
{key}:{value}
...
```
This is similar to v1 but adds `previousComicHash` and `nextComicHash`.
